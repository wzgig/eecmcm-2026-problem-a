# 项目工作要求与进度记录

更新时间：2026-05-24

## 固定工作要求

1. 每个问题单独建立文件夹，问题之间不混写结果。
2. 每一问严格按五步输出：问题分析与建模思路、详细数学建模、算法设计与编程求解、结果分析与可视化建议、论文正文撰写。
3. 每一问都要参考当前项目中的题面、附件、参考资料索引、已下载政策/论文资料，以及 `A题_建模与分析规划报告.md`。
4. 代码应尽可能模块化和可复用。公共数据读取、功率平衡、指标计算、成本计算、绘图风格应放入公共模块，后续问题直接调用。
5. 所有结果应可复现：脚本从项目文件夹读取原始附件，自动生成 CSV、图表和 Markdown 摘要。
6. 绘图按照高水平中文电力期刊风格处理：优先输出矢量 PDF/SVG，同时保留高分辨率 PNG；字体、线宽、配色、图例、网格和坐标轴应适合论文排版。
7. 对关键建模口径必须显式说明，尤其是绿电直连指标、自发自用电量、购售电互斥、吨氨成本、场景权重和储能成本折算。
8. 遇到题面表述与政策物理含义存在差异时，应保留主口径和复核口径，并在论文中解释原因。
9. 问题五的政策分析必须建立在前四问的计算现象和参考资料基础上，不能写成泛泛议论文。

## 参考资料使用原则

优先级如下：

1. 题面 `A题.pdf` 和附件1-8。
2. `A题_建模与分析规划报告.md`。
3. `参考资料/00_参考资料索引.md`。
4. 政策文件 P01、P02、P03、P08。
5. 建模论文 A01、A04、C01、C02，以及其他相关 DOI 元数据或摘要。

已确认的关键依据：

- P02 明确绿电直连的源荷匹配原则和三项比例约束：自发自用比例不低于60%，绿电占总用电量比例不低于30%，上网电量比例一般不超过20%。
- P01 进一步说明自发自用电量可按内部实际新能源发电量扣减上网电量确定，并强调小时级新能源发用电量匹配。
- P08 强调源网荷储一体化、负荷侧调节、储能和新能源就近消纳，对后续问题二至问题四有支撑作用。
- A01、A04、C01 均强调电-氢-氨系统的小时级调度、制氢/合成氨耦合、灵活运行和经济性分析，是本题建模的重要参考。
